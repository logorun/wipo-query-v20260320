## 日常监控

### 监控仪表板

**访问地址**：http://216.116.160.78:8081/

**监控项目**：

| 项目 | 正常值 | 告警阈值 |
|------|--------|----------|
| 系统状态 | ✅ 正常 | ❌ 离线 |
| Worker 实例 | 5 online | < 5 或有 offline |

- **实时监控**：监控仪表板自动刷新（30秒）
- **日常检查**：每天至少查看 1 次
- **任务高峰期**：每 2-4 小时检查 1 次

### 监控脚本示例

```bash
#!/bin/bash
# 简单的健康检查脚本

# 检查 API 服务
curl -f http://localhost:3001/health || echo "API 服务异常"

# 检查监控服务
curl -f http://localhost:8081/api/status || echo "监控服务异常"

# 检查 PM2 进程
pm2 list | grep -E "online|errored|stopped"
```

## 服务监控

### PM2 服务列表

```bash
pm2 list
```

**预期输出**：
```
┌────┬──────────────────┬─────────┬─────────┐
│ id │ name             │ status  │ memory  │
├────┼──────────────────┼─────────┼─────────┤
│ 0  │ wipo-api-test    │ online  │ ~110MB  │
│ 1  │ wipo-worker      │ online  │ ~350MB  │
│ 2  │ wipo-dashboard   │ online  │ ~65MB   │
│ 3  │ wipo-monitor     │ online  │ ~60MB   │
└────┴──────────────────┴─────────┴─────────┘
```

### 查看服务详情

```bash
pm2 show wipo-worker
```

### 查看服务日志

```bash
# 实时日志
pm2 logs wipo-worker

# 最近 100 行
pm2 logs wipo-worker --lines 100

# 错误日志
pm2 logs wipo-worker --err
```

### 监控资源使用

```bash
pm2 monit
```

### 服务管理命令

| 操作 | 命令 |
|------|------|
| 重启服务 | `pm2 restart <name>` |
| 停止服务 | `pm2 stop <name>` |
| 启动服务 | `pm2 start <name>` |
| 删除服务 | `pm2 delete <name>` |
| 重启所有 | `pm2 restart all` |
| 保存配置 | `pm2 save` |

### 扩展 Worker 实例

```bash
# 扩展到 5 个实例
pm2 scale wipo-worker 5

# 扩展到 10 个实例
pm2 scale wipo-worker 10
```

## 备份恢复

### 需要备份的内容

| 内容 | 路径 | 频率 |
|------|------|------|
| 数据库 | `/home/projects/wipo-query-v20260320/data/` | 每日 |
| 结果文件 | `/home/projects/wipo-query-v20260320/output/` | 每日 |
| 配置文件 | `/home/projects/wipo-query-v20260320/api/src/config/` | 变更时 |
| PM2 配置 | `pm2 save` | 变更时 |

### 备份脚本

```bash
#!/bin/bash
# backup.sh - 每日备份脚本

DATE=$(date +%Y%m%d)
BACKUP_DIR="/backup/wipo/$DATE"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
cp -r /home/projects/wipo-query-v20260320/data/ $BACKUP_DIR/

# 备份结果文件
cp -r /home/projects/wipo-query-v20260320/output/ $BACKUP_DIR/

# 备份配置
cp -r /home/projects/wipo-query-v20260320/api/src/config/ $BACKUP_DIR/

# 压缩
tar -czf $BACKUP_DIR.tar.gz $BACKUP_DIR
rm -rf $BACKUP_DIR

# 删除 7 天前的备份
find /backup/wipo/ -name "*.tar.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_DIR.tar.gz"
```

### 设置定时备份

```bash
# 编辑 crontab
crontab -e

# 添加每天凌晨 2 点执行备份
0 2 * * * /path/to/backup.sh >> /var/log/wipo-backup.log 2>&1
```

### 恢复数据

```bash
# 解压备份
tar -xzf /backup/wipo/20260314.tar.gz

# 恢复数据库
cp -r /backup/wipo/20260314/data/* /home/projects/wipo-query-v20260320/data/

# 重启服务
pm2 restart all
```

## 性能优化

### Worker 实例数调整

根据任务量和系统资源调整 Worker 数量：

| 场景 | Worker 数量 | 说明 |
|------|-------------|------|
| 低负载 | 3 | 日常少量任务 |
| 正常负载 | 5 | 推荐配置 |
| 高负载 | 10 | 大批量任务 |
| 最大配置 | 50 | OOM 测试上限 |

```bash
# 调整 Worker 数量
pm2 scale wipo-worker <数量>
```

### 内存优化

```bash
# 检查内存使用
free -h

# 检查各进程内存
pm2 list

# 如果内存不足，减少 Worker 数量
pm2 scale wipo-worker 3
```

### 队列优化

调整 Redis 配置（如果需要）：

```bash
# 检查 Redis 状态
redis-cli ping

# 查看 Redis 内存使用
redis-cli info memory
```
