## 常见问题

### ❓ 问题 1：Dashboard 无法访问

**症状**：
- 浏览器显示 "无法访问此网站"
- 页面加载超时

**排查步骤**：

1. **检查网络连接**
   ```
   - 确认可以访问其他网站
   - 尝试刷新页面
   - 尝试使用其他浏览器
   ```

2. **检查服务状态**
   - 访问监控仪表板：http://216.116.160.78:8081/
   - 如果监控也无法访问，联系技术支持

**解决方案**：
- 如果是网络问题，等待网络恢复
- 如果是服务问题，联系技术支持重启服务

---

### ❓ 问题 2：任务一直显示"等待中"

**症状**：
- 任务提交后长时间不处理
- 状态一直显示"等待中"

**排查步骤**：

1. **检查 Worker 状态**
   - 访问监控仪表板：http://216.116.160.78:8081/
   - 查看 "Worker 状态" 部分
   - 确认有 Worker 显示 "online"

2. **检查队列**
   - 在监控仪表板查看 "队列进度"
   - 如果"处理中"为 0，可能是 Worker 未运行

**解决方案**：
- 如果 Worker 离线，联系技术支持
- 如果队列堵塞，等待当前任务完成

---

### ❓ 问题 3：任务处理速度很慢

**症状**：
- 处理速度明显低于预期（5 品牌/2 分钟）
- 队列积压严重

**排查步骤**：

1. **检查 Worker 实例数**
   - 访问监控仪表板
   - 确认有 5 个 Worker 在线

2. **检查系统资源**
   - 查看监控仪表板的 "系统资源"
   - 如果 CPU 或内存使用率 > 80%，可能影响性能

**解决方案**：
- 如果 Worker 不足，联系技术支持
- 如果资源不足，等待当前批次完成后再提交新任务

---

### ❓ 问题 4：大量任务失败

**症状**：
- 结果文件中很多品牌显示"失败"
- 成功率明显下降

**排查步骤**：

1. **检查代理池状态**
   - 访问监控仪表板
   - 查看 "代理池状态"
   - 确认状态显示"就绪"

2. **检查网络连接**
   - 确认服务器可以访问互联网

**解决方案**：
- 如果代理池异常，联系技术支持
- 如果是 WIPO 网站问题，稍后重试

---

### ❓ 问题 5：结果文件下载失败

**症状**：
- 点击下载无反应
- 下载的文件无法打开

**排查步骤**：

1. **检查浏览器设置**
   - 确认浏览器允许下载
   - 检查下载文件夹

2. **检查任务状态**
   - 确认任务状态为"已完成"
   - 处理中的任务无法下载

**解决方案**：
- 尝试使用其他浏览器
- 等待任务完成后再下载
- 联系技术支持获取备份

---

### ❓ 问题 6：监控仪表板显示异常

**症状**：
- 监控数据不更新
- 显示"获取失败"

**排查步骤**：

1. **刷新页面**
   - 点击 "刷新" 按钮
   - 或按 F5 刷新浏览器

2. **检查 API 服务**
   - 如果持续显示异常，可能是 API 服务问题

**解决方案**：
- 刷新页面
- 联系技术支持检查 API 服务

```bash
pm2 logs wipo-dashboard
```

### API Server 日志


### Worker 日志

```bash
pm2 logs wipo-worker
```

### 监控服务日志

```bash
pm2 logs wipo-monitor
```

```bash
pm2 logs wipo-worker --lines 100
```

### 实时查看日志

```bash
pm2 logs wipo-worker

```

## 服务重启

> ⚠️ 注意：服务重启需要技术人员操作，可能导致正在处理的任务中断。

### 查看所有服务状态

```bash
pm2 list
```

### 重启单个服务

```bash
# 重启 Dashboard
pm2 restart wipo-dashboard

# 重启 API Server
pm2 restart wipo-api-test

# 重启 Worker
pm2 restart wipo-worker

# 重启监控
pm2 restart wipo-monitor
```

### 重启所有服务

```bash
pm2 restart all
```

### 停止服务

```bash
pm2 stop wipo-worker
```

### 启动服务

```bash
pm2 start wipo-worker
```

## 紧急情况处理

### 情况 1：系统完全无响应

**步骤**：
1. 尝试访问监控仪表板
2. 如果无法访问，联系技术支持
3. 技术人员会检查服务器状态并重启服务

### 情况 2：数据丢失风险

**步骤**：
1. 立即停止提交新任务
2. 下载所有已完成的结果文件
3. 联系技术支持进行数据备份

### 情况 3：代理池故障

**步骤**：
1. 检查代理服务器状态（需要技术人员）
2. 联系技术支持修复代理池
3. 代理恢复后，系统自动恢复正常

## 问题分类速查表

| 问题类型 | 可能原因 | 解决方案 |
|---------|---------|---------|
| Dashboard 无法访问 | 网络问题/服务停止 | 刷新/联系技术支持 |
| 任务不处理 | Worker 离线 | 联系技术支持 |
| 处理慢 | Worker 不足/资源不足 | 等待/分批提交 |
| 大量失败 | 代理池问题/WIPO 限制 | 联系技术支持 |
| 下载失败 | 浏览器设置/任务未完成 | 换浏览器/等待完成 |
