## 1. 设计概述

### 1.1 项目目标
构建一个功能完善的 Web 管理界面，让用户能够：
- 实时查看任务状态和进度
- 查看任务包含的商标列表
- 查看查询结果（Registered 状态的商标记录）
- 提交新的商标查询任务

### 1.2 美学方向
**科技工业风 + 实时数据感**

- **视觉语言**: 深色背景、霓虹蓝绿渐变、实时脉冲动画
- **色调**: 深蓝灰底色 (#0f172a) + 青色/翠绿数据高亮 + 琥珀色警告
- **字体**: JetBrains Mono（代码感）+ 系统默认中文字体
- **动效**: 实时数据脉冲、进度条流光、抽屉滑入滑出

## 3. 功能规格

### 3.1 实时更新机制
- **自动刷新**: 每 5 秒轮询一次任务状态
- **进度条**: 实时显示 `processed / total` 百分比
- **日志流**: 按时间顺序显示每个商标的处理状态
- **完成标记**: 已完成的商标显示绿色 ✓，进行中显示蓝色 ⏳

### 3.2 交互设计

#### 任务列表
- 点击任意任务行 → 右侧抽屉滑出
- 抽屉外点击或点击 X → 抽屉关闭
- 状态筛选下拉框 → 过滤显示特定状态任务

#### 抽屉面板
- **标签页设计**: 概览 | 结果 | 原始数据
- **结果表格**: 可排序、可筛选（按国家、状态）
- **卡片展开**: 点击表格行展开卡片详情

### 3.3 数据展示

#### 统计卡片
- 总任务数: 所有任务计数
- 已完成: status = 'completed'
- 处理中: status = 'processing'
- 失败: status = 'failed'

#### 图表
- **状态分布**: 饼图显示各状态占比
- **每日查询量**: 折线图显示最近 7 天查询趋势

#### 任务列表表格列
1. 任务ID (前8位 + ...)
2. 商标数量 (如: 10个)
3. 进度条 (可视化进度)
4. 状态徽章 (带颜色区分)
5. 创建时间 (格式化显示)
6. 操作按钮 (查看)

#### 结果表格列
1. 商标名称
2. 状态 (Registered)
3. 状态日期
4. 国家
5. 国家代码
6. 注册号
7. 是否欧盟
8. 是否欧洲
9. 操作 (展开详情)

- **框架**: 原生 HTML5 + Vanilla JavaScript
- **样式**: Tailwind CSS (CDN)

```javascript

GET /api/v1/tasks              // 任务列表
```

## 7. 性能考虑

- **虚拟滚动**: 如果结果列表很长 (>100条)

- **缓存**: 已加载的任务详情缓存
