## 系统概述

WIPO 商标采集系统是一个自动化的商标数据采集平台，主要功能：

- 📥 批量提交商标查询任务
- 🔄 自动采集 WIPO 官网数据
- 📊 生成结构化结果文件
- 🛡️ 使用代理池防止触发限制

### 系统特点

| 特点 | 说明 |
|------|------|
| 高并发 | 5个 Worker 并行处理 |
| 代理轮换 | 253个 SOCKS5 代理，自动轮换 |
| 自动重试 | 失败自动重试，熔断保护 |
| 实时监控 | 监控仪表板实时显示状态 |

## 组件说明

### 1. 🌐 Dashboard（仪表板）

**功能**：Web 管理界面，用于提交任务和查看结果

**访问地址**：http://216.116.160.78:8080/

**主要功能**：
- 上传 Excel 文件（品牌列表）
- 查看任务进度
- 下载结果文件
- 查看历史记录

---

### 2. 🔌 API Server（API 服务）

**功能**：后端 API 服务，处理任务请求

**访问地址**：http://216.116.160.78:3001/

**主要功能**：
- 接收任务提交请求
- 管理任务队列
- 提供任务状态查询
- 生成结果文件

**进程名称**：`wipo-api-test`

---

### 3. 👷 Worker（工作进程）

**功能**：执行实际的采集任务

**实例数量**：5 个（并行处理）

**主要功能**：
- 从队列获取任务
- 使用代理访问 WIPO 网站
- 解析网页数据
- 保存结果到数据库

**进程名称**：`wipo-worker`

**处理能力**：约 5 个品牌/2 分钟

---

### 4. 📊 Monitor（监控服务）

**功能**：实时监控系统运行状态

**访问地址**：http://216.116.160.78:8081/

**主要功能**：
- 显示系统状态
- 监控 Worker 实例
- 显示代理池状态
- 显示系统资源使用
- 异常告警

**进程名称**：`wipo-monitor`

---

### 5. 🌐 Proxy Pool（代理池）

**功能**：提供 IP 代理服务，防止触发限制

**代理服务器**：23.148.244.2

**代理数量**：253 个 SOCKS5 代理

**端口范围**：10000 - 10252

**轮换策略**：round-robin（轮询）

**冷却时间**：30 秒

---

### 6. 💾 Redis（队列服务）

**功能**：任务队列和缓存

**访问地址**：localhost:6379

**主要功能**：
- 存储任务队列
- 缓存临时数据
- 代理状态存储

## 数据流程

### 任务提交流程

```
1. 运营人员上传 Excel → Dashboard
2. Dashboard 发送请求 → API Server
3. API Server 创建任务 → Redis 队列
4. Worker 从队列获取任务
5. Worker 使用代理访问 WIPO
6. Worker 解析数据并保存
7. 运营人员下载结果 ← Dashboard
```

### 代理使用流程

```
1. Worker 获取任务
2. ProxyManager 选择下一个代理（round-robin）
3. Worker 通过代理访问 WIPO
4. 代理服务器转发请求到 WIPO
5. WIPO 返回数据
6. 代理服务器返回数据给 Worker
7. Worker 处理数据
```

## 常见问题

### Q: 为什么需要代理池？

A: WIPO 网站对频繁访问有限制，使用代理池可以：
- 分散请求来源 IP
- 避免触发反爬机制
- 提高采集成功率

### Q: 5 个 Worker 是什么意思？

A: 系统同时运行 5 个工作进程，可以并行处理 5 个商标查询，提高处理速度。

### Q: 代理轮换是什么？

A: 每次查询使用不同的代理 IP，253 个代理轮流使用，确保每个 IP 不会过于频繁访问。
